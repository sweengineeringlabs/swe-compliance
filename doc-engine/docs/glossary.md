# Glossary

> **Audience**: Developers, architects, project stakeholders

**ADR** - Architecture Decision Record: a document capturing a significant architectural decision, its context, and consequences.

**CLI** - Command-Line Interface: the terminal-based user interface for running doc-engine.

**SDLC** - Software Development Lifecycle: the phased process for planning, creating, testing, and deploying software.

**SEA** - Stratified Encapsulation Architecture: an architectural pattern organizing code into layered modules with clear encapsulation boundaries.

**TL;DR** - Too Long; Didn't Read: a brief summary section placed at the top of long documents.

**TOML** - Tom's Obvious Minimal Language: the configuration file format used for doc-engine rule definitions.

**W3H** - Who, What, Why, How: a structural template for documentation hub pages ensuring key questions are answered.

**YAML** - YAML Ain't Markup Language: a human-readable data serialization format used for structured spec files.
