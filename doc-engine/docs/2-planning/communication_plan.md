# Communication Plan

**Audience**: Developers, project stakeholders

## Channels

| Channel | Purpose | Frequency |
|---------|---------|-----------|
| GitHub Issues | Bug reports, feature requests | As needed |
| GitHub PRs | Code review, discussion | Per change |
| CHANGELOG.md | Release notes | Per release |
| README.md | Project overview, quick start | Updated as needed |

## Stakeholder Updates

| Audience | Format | Content |
|----------|--------|---------|
| Users | CHANGELOG.md | New checks, breaking changes, fixes |
| Contributors | CONTRIBUTING.md | Development workflow, coding standards |
| Security | SECURITY.md | Vulnerability reporting process |

See [architecture.md](../../docs/3-design/architecture.md) for system design context.
